# 深層借鏡研究：nuwa-skill 的設計思維 → ASP

本報告為**評估交付物**，非實作。深層研究 `alchaincyf/nuwa-skill`（女媧.skill，persona 蒸餾品類）的底層設計思維，對抗式判斷哪些優勢能遷移到 ASP（行為憲法 + 機械強制力，**不同品類**），並扣回 P0–P4 改造序列。所有「借鏡動作」皆為提案，需經 ADR 流程核准後方可落地。承接反思報告 P1 對 nuwa 的初步引用（`docs/research/2026-06-22-external-benchmark-reflection.md`），把它從一句話深化成可執行判準。

## 0. 方法論

12-agent 對抗式 workflow：(1) 平行讀 nuwa 一手來源（`SKILL.md`、`references/extraction-framework.md`、`README_EN`、Darwin skill、實際 example）；(2) 蒸餾「可遷移的底層思維」並剔除行銷話術；(3) 六條優勢逐一 grounded 在 ASP 實際檔案做對抗式遷移判斷。每條附「能否遷移 / 具體動作 / 反例風險」。

## 1. 先破三個迷思（深層借鏡的起點）

一手查證戳穿了二手敘事——借鏡前先剝行銷層：

| 流傳說法 | 一手查證結果 |
|---------|------------|
| examples-first（範例擺最前） | SKILL.md 其實 **instructions-first**；真身是「每條抽象規則旁就近掛一個具體範例/反例」的**密度紀律** |
| 跨 50+ runtime 可攜是 nuwa 優勢 | 那是**站在開放 Agent Skills 協定上**的紅利，非 nuwa 發明；可學的是「自包含、複製整個目錄就能跑」 |
| 兩月兩萬 star、無推廣 | 一手 README **查無此句**，是外圍成長敘事，非設計優勢 |

## 2. 蒸餾的底層思維（按可遷移性排序）

**最高遷移性三條（跳脫 persona 品類）：**

1. **可證偽 + 失效邊界**——每個斷言釘在「≥2 個跨域證據」上，且強掛 `局限/失效條件` 欄。沒有局限欄的論點在這套規則下根本不成立（Popper 可證偽性落到 prompt 工程）。
2. **改的 agent ≠ 評的 agent，且只進不退（棘輪）**——作者與評分者職能分離（實證 LLM 自評僅 46.4% 準）＋ 換評委防錨定 ＋ 改進後嚴格更優才保留、否則 git 自動回滾。**遷移價值最高。**
3. **HOW 而非 WHAT**——蒸餾「可運行的生成器」而非「可拼貼的查找表」。判準：「刪掉名字還認得出是誰的思維方式嗎？」測泛化而非保真。

**誠實工程一體兩面：** (4) 矛盾是特徵不是 bug；(5) 誠實標註的 60 分勝過看似完美編造的 90 分（把「不知道」做成第一級欄位，校準比準確率更致命）。

**好技法但偏場景綁定：** (6) 量化主觀（句式指紋）；(7) 雙入口路由（明確需求直接執行／模糊需求先診斷反推）。

**不要過度遷移的：** persona 體驗細節（免責只說一次、用角色方式猶豫）、模板欄位（智識譜系/身份卡）、行銷外皮（一行 npx、純 markdown 至上）。

## 3. 最深的發現（貫穿全部六條映射）

> **ASP 不需要「引進」nuwa 的好點子——它早已獨立長出大半胚胎，只是停在「散文自律」沒升成「機械結構」。**

| nuwa 優勢 | ASP 已自發長出的胚胎 |
|----------|----------------------|
| 職能分離（評者≠作者） | `reality-checker` 預設 NEEDS_WORK、不信自評、read-only（踩過給寫入權竄改 ADR 才收斂） |
| 只進不退棘輪 | G6 `health_score_regression: false`、G4 coverage delta<0 拒絕 |
| 失效條件 | `.asp-fact-check.md` 的「再驗證條件」欄（FC-001 已有） |
| 誠實邊界 | ADR-020 整份（fail-open＝「強制力讓位可用性的誠實殘留」）、ADR-003 cancelled |

借鏡的本質＝把胚胎「**從個人自律升格為結構義務**」——這恰是 ASP 框架的本職（把散文規範固化成機器可讀約束）。引力同向，非外來移植。

## 4. 唯一的硬約束（翻譯規則）

> **nuwa 的每個點子，遷移時都必須從「誠實標註」翻譯成「機械判定」。**

nuwa 的執行者是**人**（不會被 compact，靠思維內化可行）；ASP 的執行者是**會被壓縮的 AI**（散文義務在長對話壓縮後最先蒸發——CLAUDE.md「過程義務速查」整段為對抗此而生）。一個寫成自然語言的「失效條件」欄，hook 讀不懂、執行不了，就退化成 ADR-020 點名的「壓縮後蒸發的散文義務」。**即使最該抄的 #1/#2，都得把「誠實」翻成「機械」。**

## 5. 可執行借鏡（去重後扣回 P2–P4）

| 動作 | 落點 | 內容 |
|------|------|------|
| **★ 治理複雜度預算棘輪（→ ADR-022）** | **P3** | `rule-stats`（ADR-018）從單向「90 天零命中砍除」升級為**命中率 × 可推導性**雙軸；`profiles.total_lines` baseline（凍結 5177）設**只進不退硬上限**，超標 CI red 除非附 ADR 認列；評分由獨立 reality-checker 跑；鐵則/Iron Rule 豁免；保留 ADR 認列逃生門。**最高槓桿、ASP 自己吃狗糧。** |
| ADR/SPEC 模板加 `失效條件 / 退場條件` 欄 | 小 ADR | nuwa #1 制度化；`asp-gate G1` 把「空欄」列 FAIL——但**只在引入新強制力的重量級 ADR 觸發**，輕量改動豁免（否則 ADR 通膨） |
| `.asp-fact-check.md` 加「失效條件」欄 | 零成本 | 對標 PR #27「查到了但不知 API 欄位邊界、誤判三次」 |
| 職能分離從散文升為**層硬約束** | **P2 拆層** | 「reality-check 由獨立 read-only subagent、且非產出該碼的同一 context」寫死 |
| 切斷 `global_core.md` 對 hook 的 **9 處硬編** | **P2** | 知識層只描述 WHAT、強制力層描述 HOW → 雙層解耦，知識層才真能獨立複製 |
| `portability: {full｜advisory-only｜host-bound}` 契約欄 | **P4** | 把可攜性從目標降為**誠實欄位**；P4 交付＝「誠實標清楚哪些跨 runtime 就沒牙」 |
| 能力 fixture（違規→被擋的可復現 demo） | **P2/P4** | README quickstart 升級成 repo 內 fixture，當可攜層 conformance suite——**須覆蓋失效邊界非 happy-path、選擇性非全員義務、走官方 plugin 不自製 runner** |

## 6. 千萬別照搬（會直接弄壞 ASP）

- **「只做一件事 / 殘酷縮放」→ 反向有害。** 憲法本質覆蓋一組相互制衡義務；ASP 的病是**機制過重**（雙文件 + 29KB installer + mono-router），非功能過多。診斷錯藥就錯。
- **「HOW 而非 WHAT」用在鐵則層 → 違反立身之本。** 硬強制來自**不可泛化的機械具體性**（deny 一個具體指令字串）。把鐵則蒸餾成「思維」＝開逃逸路徑。
- **全自動 Darwin 自我改寫 → 撞四鐵則**（ADR 須 Accepted 才實作、破壞性須人類確認）。只抄「評者≠作者 + 只進不退」這對**結構**，不抄「自動」。
- **「矛盾是特徵」用在 gate 設計 → bug 不是 feature**（規則互相衝突要消解，不是保留）。
- **「純 markdown 至上 / 一行分發」→ 會把 ASP 還原成沒牙規範。** hook 牙齒、88KB 編譯產物（ADR-016 故意反可攜）、installer 強制力管線是 **essential 且故意不可攜**；可攜性與「擋得住」在 ASP 裡是**對立目標**。
- **每個採納過 ADR-010 摩擦評估。** 用機器可讀客觀指標（行數/exit code/命中數）取代 nuwa 的「多 LLM judge 投票」，才合 ASP 機械強制力本質。

## 7. 收斂

nuwa 給 ASP 的最大價值**不是新功能，是一面鏡子 + 一條翻譯規則**：ASP 已自發長出三個最好的胚胎（職能分離、棘輪、失效條件），借鏡＝把它們**從散文自律升成機械結構**，全程把「誠實標註」翻成「機械判定」。**最高槓桿的單一動作＝ P3 的 `ADR-022 治理複雜度預算棘輪`**——同時吸收 #1/#2/#3，且正是 ASP 該示範的「自己吃狗糧」。

> 後續：本報告為 P3 的依據，`ADR-022` 將引用之；其餘借鏡動作（失效條件欄、雙層解耦、portability 契約、能力 fixture）分屬 P2/P4，各自走 ADR。
