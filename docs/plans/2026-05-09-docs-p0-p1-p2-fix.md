> **Last Updated:** 2026-05-09 | **Status:** Current | **Audience:** Developer

ASP v4.0 加入 JSONL append-only telemetry，用於量測 gate 效果、bypass 頻率、session blocker 次數，提供改善 ASP 規則的數據基礎。

---

## 快速開始

```bash
make asp-telemetry-collect   # 記錄當前 session 事件到 .asp-telemetry.jsonl
make asp-telemetry-report    # 產出最近 7 天的週報（stdout）
make asp-telemetry-prune     # 封存 90 天以前的記錄
```

---

## 事件類型

所有事件寫入 `.asp-telemetry.jsonl`（每行一個 JSON 物件）。

| 事件類型 (`event_type`) | 何時觸發 | 關鍵欄位 |
|------------------------|---------|---------|
| `session_start` | session-audit.sh 執行時 | `asp_version`, `blockers`, `warnings`, `profile_type` |
| `gate_result` | `/asp-gate G1-G6` 執行後 | `gate_id`, `status` (PASS/FAIL/BLOCKED), `findings` |
| `bypass_recorded` | `make asp-bypass-record` | `skill`, `step`, `reason`, `actor` |
| `skill_invoked` | 任一 `asp-*` skill 執行後 | `skill_name`, `trigger`, `result` |

---

## JSON 事件格式

每個事件的基本欄位：

```json
{
  "ts": "2026-05-09T06:23:01.992841+00:00",
  "event_type": "session_start",
  "asp_version": "4.0.0",
  "data": {
    "blockers": 0,
    "warnings": 1,
    "profile_type": "system"
  }
}
```

---

## 如何解讀 Weekly Report

```bash
make asp-telemetry-report
# 或指定天數：
python3 .asp/scripts/telemetry/report.py --days 14
```

報告輸出包含：

- **Gate 通過率：** 哪些 gate 從未失敗（候選簡化）
- **Bypass 熱點：** 哪個 skill 被 bypass 最多次（候選強化）
- **Session Blocker 趨勢：** 哪些 blocker 反覆出現
- **未觸發 Skill：** 哪些 skill 從未被呼叫（候選移除）

> **判讀原則：** 一個 gate 從未失敗 ≠ 它沒用；可能是它真的防住了問題。
> 用 bypass 率（bypass / total invocations）而非絕對次數來判斷 skill 是否太嚴。

---

## 資料存放

```
.asp-telemetry.jsonl    # 當前月份事件（append-only）
.asp-telemetry-YYYY-MM/ # 封存的月份資料（make asp-telemetry-prune 建立）
```

Telemetry 是 **local-only**，不上傳，已加入 `.gitignore`。

## 設計原則（來自 ADR-004）

- **零 runtime 依賴：** 純 Python stdlib，不需安裝套件
- **Silent failure：** telemetry 寫入失敗不影響 ASP 正常運作
- **Grep-able：** `grep "event_type.*gate_result" .asp-telemetry.jsonl | jq .`
- **Archive-friendly：** `prune.py` 每月封存，不刪除歷史資料
