## Findings

### F-001（LOW）：blockquote 引用一個不存在的 Makefile target
- **嚴重度**：LOW
- **描述**：blockquote 寫「Makefile 的 `make diagram`（舊 `architecture.html` 產生器）」，但 Makefile 中查無此 target。ADR-008 第 80 行決策段落也提及此 target 為「保留 deprecated alias」，惟 Makefile 從未建立它。
- **影響**：讀者可能誤以為曾存在一個舊 `make diagram` target，實際上並無。
- **建議**：可將措辭改為「Makefile 中目前不存在 `make diagram` target（該 target 僅在 ADR 決策文字中規劃，尚未落地），故不在本 skill 實作範圍」，消除歧義。此為非阻擋性建議。

### F-002（INFO）：SPEC-005 Draft (Deferred) 未設定追蹤 ticket/milestone
- **嚴重度**：INFO
- **描述**：SPEC-005 狀態為 `Draft (Deferred)`，全部實作列 PENDING，但 SPEC-005 本身未標明「何時升級至 Stage 2」的觸發條件或 milestone。blockquote 引用 SPEC-005 消除審計疑慮，但缺少時間邊界可能讓未來審計員難以判斷是「延後中」還是「永久擱置」。
- **影響**：長期維護可見性輕微降低。
- **建議**：可在 SPEC-005 或 blockquote 補一句觸發條件（例如「待 ASP 治理需求成熟後再排入 roadmap」），讓語意更完整。非阻擋性。
