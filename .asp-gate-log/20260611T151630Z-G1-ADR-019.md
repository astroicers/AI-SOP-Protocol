## G1 品質逐項檢查

### 1. Context 清晰度

**評定：通過。**  
背景段落明確說明：（a）Iron Rule B 目的（append-only 稽核完整性）；（b）TD-002 已完成的修正（HWM sidecar）；（c）TD-002 自身文件化的兩項殘留漏洞（等量替換、HWM sidecar 竄改），連同程式碼位置（session-audit.sh:91-93）一起引用。驅動因素清晰：「改一個數字即可無痕抹除鐵則稽核」是明確的外部威脅動機。

### 2. Alternatives 評估深度

**評定：通過（品質良好）。**  
四個選項均有實質評估，非走形式：

- **選項 A**（採用）：列出優缺點與風險，包含「知道機制可重算假 chain」的誠實風險承認。
- **選項 B（HMAC）**：否決理由具體——「key 存 `~/.claude` 下，與 AI/操作者同等可讀，防護等於零」，指出在本地 agent 威脅模型下 key 不構成秘密，因此 HMAC 製造假安全感。
- **選項 C（git 錨定）**：否決理由為「與 TD-002 local-only 決策張力大、每筆 bypass 強制一次 commit 流程沉重、超出收尾範圍」，保留為後續獨立 ADR。
- **選項 D（維持現狀）**：否決理由為鐵則完整性持續依賴脆弱不變量。

### 3. Decision 明確性

**評定：通過。**  
Decision 段明確選擇選項 A，且誠實界定威脅模型（tamper-evidence 非 tamper-proof），清楚說明能偵測什麼、不聲稱防禦什麼，並解釋相對其他選項的選擇理由。

### 4. Consequences 真實 trade-off

**評定：通過，帶一條 WARNING。**  
Consequences 包含正面（偵測等量替換與 HWM 竄改）與負面（schema 變更、遷移工作、寫入端修改）。`tech-debt: MED` 標記誠實承認不防完整重算。

**WARNING（F-01）**：Consequences 未提及 canonical 格式穩定性失敗的風險。SPEC-012 邊界條件中指出「canonical 必須穩定（key 排序 + 緊湊），否則同內容算出不同 hash → 假陽性」，但 ADR 本文的 Consequences 並未列出此風險 — 若 canonical 實作存在平台差異（bash vs Python 排序行為、空白/unicode 處理），會導致正常 session 觸發 chain 驗證 BLOCKER（假陽性），比等量替換更頻繁發生。

### 5. Success Metrics 可量化性

**評定：通過。**  
成功指標表格含四項可量化目標（100%/0 false positive/全綠），驗證方式具體（`test_iron_rule_b_hashchain.sh`、`make test`），檢查時間點明確（實作完成時）。

## 核心主張的對抗式威脅模型驗證

### 主張：「hash chain 是 tamper-evidence 非 tamper-proof」

**評定：站得住，且是本 ADR 最有價值的品質。**

ADR 明確承認：「知道機制、且對所有檔有寫入權的攻擊者可完整重算假 chain」— 在純本地、無外部信任錨、AI 自身可讀寫一切的環境下，密碼學上確實不可能防禦。這個限制說明是誠實的，與 SPEC-012 一致。

### 主張：選項 B（HMAC）是 security theater

**評定：理由成立，但論述稍微跳躍。**

論證鏈：「key 存 `~/.claude`」→「AI/操作者同等可讀」→「key 不構成秘密」→「防護等於零」。這個推論在「本地 agent 自我竄改」威脅模型下成立。但 ADR 未明說一個前提：**本 ADR 的主要威脅行為者是「本地 AI agent 的自我修改」，而非「外部攻擊者入侵後的痕跡清除」**。若對抗的是外部人類攻擊者（而非 AI agent 自身），HMAC with local key 提供一定阻力（對手需讀取 key 後才能偽造）。ADR 的 security theater 結論在 AI-as-threat-actor 模型下正確，但威脅行為者的明確界定在 ADR 本文中是隱含而非顯式的。

（注意：ADR-002 有完整 STRIDE 威脅模型，可作為補強參考。）

## 總評

ADR-019 是一份**誠實且架構清晰**的決策文件。最值得肯定的是威脅模型的誠實性——主動承認 hash chain 在純本地環境的密碼學局限，避免過度承諾安全性，這比 HMAC 的假保證更有工程誠信。四個選項均有實質評估，成功指標可量化，與 SPEC-012 配套完整。

Findings 主要為文件補強（canonical 假陽性風險遺漏、威脅行為者未顯式界定）而非架構缺陷，不阻擋通過。

**狀態 Draft = 實作需等人類定案（升 FIRM 須附 Verification Evidence）。G1 結論：GATE_PASS_WITH_WARN。**
