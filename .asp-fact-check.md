## FC-001 — Claude Code `settings.local.json` 的 deny 合併行為

- **日期**：2026-06-08
- **觸發**：ADR-011（將 ASP 動態 deny 隔離至 settings.local.json）的核心假設
- **查證方式**：claude-code-guide subagent 對照官方文件（code.claude.com/docs/en/settings、/permissions）
- **結論（已確認）**：
  1. `.claude/settings.local.json` 是 Claude Code 自動載入的設定 scope。✅
  2. `permissions.deny` / `permissions.allow` **跨 scope 合併**；評估順序 **deny → ask → allow**，**任一 scope 的 deny 即阻擋**（deny-first，與檔案優先序無關）。✅
  3. 優先序：Managed > Command-line > **Local (settings.local.json)** > Project (settings.json) > User。✅
  4. `settings.local.json` **預設 gitignored**，定位為本地 / 不提交的 per-project override。✅
  5. Hook 寫入 settings.local.json 的 deny 同 session 自動 reload 生效，並持續至下次 session。✅
- **對 ADR-011 的意涵**：把 ASP 動態 deny 寫到 settings.local.json，其強制力與寫在 settings.json **等價**（deny-first、scope-agnostic），同時獲得 gitignored + 本地隔離 → 解決 TD-005 殘留。
- **再驗證條件**：Claude Code 若變更 settings 載入或 deny-merge 語意，FC-001 失效，須重審 ADR-011。
